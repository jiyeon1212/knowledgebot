# 요구사항: 프로젝트 기반 구조화 검색

## 배경

현재 Slack 봇은 자연어 입력 → AI 의도 분류 → 키워드 검색 방식으로 동작한다.
문제점:
- AI 키워드 추출이 부정확 (OR/AND 규칙 혼동)
- 동일한 키워드가 플랫폼별로 다르게 해석됨 (Drive는 공백을 OR로, Gmail은 AND로 처리)
- 키워드 매칭만으로는 문서명에 프로젝트명이 없는 관련 문서를 찾지 못함

## 요구사항

### 1. 구조화된 검색 폼 (Slack Modal)

`/검색` 슬래시 커맨드 입력 시 Slack Modal 폼을 표시한다.

입력 필드:
- **프로젝트명** (텍스트, 필수)
  - 콤마(,)로 구분 시 OR 검색 (예: "kbtf, k-btf" → 두 프로젝트 동시 검색)
  - 같은 프로젝트의 다른 표기도 콤마로 지원 (예: "kbtf, k-btf")
- **카테고리** (선택, 필수)
  - 사업: 제안서, 기안서, 견적서, 계약서, 발주서, 검수서 등 사업 관련 문서
  - 개발: API, SDK, 연동 가이드, 테스트, 개발 문서 등 개발 관련 문서
- **기간** (선택, 선택사항)
  - 디폴트: 전체
  - 옵션: 최근 1개월 / 최근 3개월 / 최근 6개월 / 직접 입력

### 2. 프로젝트 기반 검색 (스페이스/폴더 기반)

프로젝트명을 입력받으면 키워드 매칭이 아닌 **스페이스/폴더/프로젝트 단위**로 검색한다.

- **Confluence**: 프로젝트명으로 상위 페이지를 찾고 (`title~"프로젝트명"`), 해당 페이지의 모든 하위 페이지를 조회 (`ancestor={pageId}`)
  - 부서별로 Confluence 구조가 다를 수 있으므로 계층 구조에 의존하지 않음
  - `ancestor` 검색은 모든 하위 계층을 포함함
- **Drive**: 프로젝트명으로 폴더를 찾고, 해당 폴더 내 모든 파일을 조회
- **Jira**: 프로젝트명으로 프로젝트를 찾고, 해당 프로젝트의 이슈를 조회
- **Gmail**: 프로젝트명 + 카테고리별 보조 키워드로 검색

### 3. AI 카테고리 필터링

스페이스/폴더 기반으로 가져온 전체 문서에서 AI가 카테고리(사업/개발)에 맞는 문서만 필터링한다.

- Confluence/Drive: 전체 문서를 가져온 뒤 AI가 카테고리 기준으로 필터링
- Gmail: 검색 시 카테고리별 보조 키워드를 사용하므로 추가 필터링 최소화

### 4. 기존 자연어 검색 유지

- 기존 DM 자연어 검색은 그대로 유지
- `/검색` 폼과 DM 자연어 검색이 병렬로 동작

### 5. 플랫폼별 쿼리 최적화

각 플랫폼 API의 문법 차이를 한 곳(쿼리 변환 레이어)에서 관리한다.

| 서비스 | 콤마(OR) 처리 | 공백 처리 |
|---|---|---|
| Gmail | `A OR B` | Gmail 자체 AND |
| Drive | 각 단어별 `fullText contains` OR | 각 단어별 OR (현재 동작) |
| Confluence | `text~"A" OR text~"B"` | 통째로 전달 |
| Jira | `summary~"A" OR summary~"B"` | 통째로 전달 |
